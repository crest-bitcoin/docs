## Endpoint

```
GET https://crestgateway.up.railway.app/meta-api/v1/quote/get-quote
```

The Crest Meta API aggregates on-chain liquidity sources and relayers to provide the best executable quote for a swap or bridge. The response returns every provider that produced a quote plus the recommended provider with the full list of actions (transactions or signatures) your app must perform.

## Authentication

Include your project key in the request header:

```
X-Api-Key: <YOUR_API_KEY>
```

## Query Parameters

<ParamField query="sender" type="string" required>
  Wallet address that will send the input tokens. Must be a valid Ethereum address.
</ParamField>

<ParamField query="receiver" type="string" required>
  Address that should receive the output tokens. Can match `sender` for same-wallet swaps.
</ParamField>

<ParamField query="inputToken" type="string" required>
  Symbol or contract address of the input (source) token.
</ParamField>

<ParamField query="outputToken" type="string" required>
  Symbol or contract address of the output (destination) token.
</ParamField>

<ParamField query="inputChain" type="number" required>
  Chain ID of the source network (e.g., `1` for Ethereum, `42161` for Arbitrum).
</ParamField>

<ParamField query="outputChain" type="number" required>
  Chain ID of the destination network.
</ParamField>

<ParamField query="inputAmount" type="string" required>
  Amount of the input token in the smallest unit (wei for ETH-based tokens).

  Example: `"1000000000000000000"` for 1 WETH.
</ParamField>

## Example Request

```bash
curl --location 'https://crestgateway.up.railway.app/meta-api/v1/quote/get-quote?sender=0xF2Cb17DD766D8522E7eD0e265f761219e1249893&receiver=0xF2Cb17DD766D8522E7eD0e265f761219e1249893&inputToken=0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2&outputToken=0xaf88d065e77c8cC2239327C5EDb3A432268e5831&inputChain=1&outputChain=42161&inputAmount=1000000000000000000' \
  --header 'X-Api-Key: key_test001'
```

## Response

<ResponseField name="providerName" type="string">
  Provider that produced the recommended quote.
</ResponseField>

<ResponseField name="inputAmount" type="string">
  Amount of the input token in base units for the selected quote.
</ResponseField>

<ResponseField name="outputAmount" type="string">
  Estimated output amount in base units.
</ResponseField>

<ResponseField name="inputToken" type="string">
  Symbol or contract address of the input token.
</ResponseField>

<ResponseField name="outputToken" type="string">
  Symbol or contract address of the output token.
</ResponseField>

<ResponseField name="inputChain" type="string">
  Source chain ID (stringified).
</ResponseField>

<ResponseField name="outputChain" type="string">
  Destination chain ID (stringified).
</ResponseField>

<ResponseField name="steps" type="array">
  Ordered actions required to execute the quote.

  <Expandable title="step object properties">
    <ResponseField name="description" type="string">
      Optional human-readable explanation of the step.
    </ResponseField>

    <ResponseField name="type" type="string">
      Step kind: `"transaction"` or `"signature"`.
    </ResponseField>

    <ResponseField name="tx" type="object">
      Transaction payload (present for `transaction` steps).

      <Expandable title="tx properties">
        <ResponseField name="data" type="string">
          Encoded calldata.
        </ResponseField>

        <ResponseField name="to" type="string">
          Target contract address.
        </ResponseField>

        <ResponseField name="value" type="string">
          ETH value (wei) to send with the transaction.
        </ResponseField>

        <ResponseField name="gas" type="string">
          Recommended gas limit or gas units.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="message" type="string">
      Base64-encoded payload that must be signed for `signature` steps.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="errorMessage" type="string">
  Null when the quote succeeded, otherwise contains provider-specific errors.
</ResponseField>

<ResponseField name="allProviders" type="array">
  Quotes returned by every upstream provider (including the recommended one).

  <Expandable title="provider quote properties">
    <ResponseField name="providerName" type="string">
      Provider identifier.
    </ResponseField>

    <ResponseField name="inputAmount" type="string">
      Input amount considered for this provider.
    </ResponseField>

    <ResponseField name="outputAmount" type="string">
      Estimated output amount for this provider quote.
    </ResponseField>

    <ResponseField name="inputToken" type="string">
      Input token symbol or address.
    </ResponseField>

    <ResponseField name="outputToken" type="string">
      Output token symbol or address.
    </ResponseField>

    <ResponseField name="inputChain" type="string">
      Source chain ID.
    </ResponseField>

    <ResponseField name="outputChain" type="string">
      Destination chain ID.
    </ResponseField>

    <ResponseField name="steps" type="array">
      Provider-specific execution steps (structure matches the root `steps`).
    </ResponseField>

    <ResponseField name="errorMessage" type="string">
      Error from the provider, if any.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Codes

| Code | Description |
|------|-------------|
| `INVALID_PARAMETERS` | Request parameters are invalid or missing |
| `INVALID_TOKEN` | Token address or symbol is not recognized |
| `TOKEN_NOT_FOUND` | Token does not exist on the specified chain |
| `UNSUPPORTED_CHAIN` | Blockchain network is not supported |
| `INSUFFICIENT_LIQUIDITY` | No available liquidity for the trading pair |
| `AMOUNT_TOO_LOW` | Swap amount is below the minimum threshold |
| `AMOUNT_TOO_HIGH` | Swap amount exceeds maximum threshold |
| `RATE_LIMIT_EXCEEDED` | Too many requests, please slow down |
| `INTERNAL_ERROR` | Internal server error, please try again |

## Notes

<Note>
**Timing Fields**: Internal fields like `elapsedAll`, `startTime`, `endTime`, and `estimatedTime` are experimental signals that may appear in sandbox environments but should be ignored in production integrations.
</Note>

<Note>
**Step Execution**: Execute the returned `steps` sequentially using a Web3 library (ethers.js, viem, wagmi, etc.). Transactions and signatures should be prompted in the same order they appear.
</Note>

<Warning>
**Slippage & Reverts**: Provider transactions already encode the required slippage controls. Any modification to calldata may remove those protections and cause the swap to revert.
</Warning>

<Tip>
**Cross-Chain Swaps**: When `inputChain` and `outputChain` differ, expect a bridge or relayer transaction in the steps array. Delivery time depends on the selected provider.
</Tip>
